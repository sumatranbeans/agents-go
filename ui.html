Agents: Go

        
Your autonomous Claude Code agent scheduler for macOS

        
Schedule and run Claude Code CLI agents across multiple projects on autopilot. This tool is exclusively for Claude Code — each agent gets a fresh claude session, reads its own project context, does its work, and exits cleanly.

        

          http://localhost:3847
          
        

      


      
      

        

          
        

        
Add Projects

        
Point each project to the local directory where Claude Code operates. This is where your agents will cd into before running.

        
You can manage multiple projects simultaneously, each with its own agents and independent schedules.

      


      
      

        

          
        

        
Add Agents

        
Create agents within each project. Give them a name and an invocation prompt (the instruction they receive when summoned).

        
You need to set up your agent personas and tasks in your codebase before scheduling them here. Every session is completely fresh with no state or memory carrying over between runs.

      


      
      

        

          
        

        
Set Schedule

        
Configure how often each agent runs:

        
    
          
  • Days of the week
    • 
          
  • Hours of the day (AM/PM, your local time)
    • 
          
  • Interval: every 15, 30, 45, or 60 minutes within active hours
    • 
        

      


      
      

        

          
        

        
Watchouts

        
    
          
  • Tokens: Each session burns API tokens. Start with 60-min intervals and monitor your Anthropic usage dashboard.
    • 
          
  • Permissions: Claude Code requires a one-time permissions acceptance on first run. It remembers permanently.
    • 
          
  • Storage: No state persists between runs. Logs auto-clean after 14 days.
    • 
          
  • Stuck Agents: Use "Kill All Running" on the Dashboard.
    •
